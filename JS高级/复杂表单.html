表单验证

为了防止用户的误输入，实际生产环节中，我们都需要对表单进行验证。常见的input控件有text,radio,checkbox等，HTML5新加了如下类型的input 控件email,url,number,range,datepickers,search,color

Input type 用途 说明
email 电子邮件地址
url 网页URL文本框
number 数值的输入域 max:规定允许的最大值 min:规定允许的最小值 step:规定合法的数字间隔（如果 step="3"，则合法的数是 -3,0,3,6 等）
range 特定值的范围的数值，以滑动条显示 可设置 max min step属性，用法和上面一致
search 用于搜索引擎，比如在站点顶部显示的搜索框 与普通文本框用法一样，只不过这样更语文化
color 颜色选择器

使用合适的 input type 能帮我们事半功倍的完成很大一部分的表单验证，但是在某些需求下，我们还是需要使用一些其他属性来完成。

pattern 属性

pattern 属性规定用于验证 input 域的模式（pattern）。模式（pattern） 是正则表达式

pattern 属性适用于以下类型的 标签：text, search, url, telephone, email 以及 password。

<form action="/">
username: <input type="text" name="username" pattern="^[a-zA-Z_]\w{5,13}$"/>
<input type="submit" />
</form>

required 属性

required 属性规定必须在提交之前填写输入域（不能为空）。

required 属性适用于以下类型的 标签：text, search, url, telephone, email, password, date pickers, number, checkbox, radio 以及 file。

<form action="/">
Name: <input type="text" name="username" required="required" />
</form>

自定义表单验证

一个常见的表单验证需求是判断用户两次输入的密码是否一致，html5默认的表单验证无法做到，这时候就需要我们使用自定义表单验证

我们可以通过 inputElement.validity 获取一个input的验证状态，它有如下几个字段

  1. valueMissing
  2. typeMismatch
  3. patternMismatch
  4. tooLong
  5. rangeUnderflow
  6. rangeOverflow
  7. stepMismatch
  8. customError

其中只要有一个值为true，则最终的验证结果为验证失败 前7个为浏览器默认提供的表单验证，最后一个是我们需要手动设置的

我们可以使用 inputElement.setCustomValidity("两次输入的密码不一致") 来设置第八项为true。使用 inputElement.setCustomValidity("")第八项为false

下面给出判断用户两次输入密码是否一致的实现

<form action="/">
password: <input type="text" id="password" oninput="passwordInput()"/>
password repeat: <input type="text" id="password-repeat" oninput="passwordRepeatInput()"/>
<input type="submit" />
</form>
<script>
    var password;
    function passwordInput(){
        password = document.getElementById("password").value
    }
    function passwordRepeatInput(){
        var input=document.getElementById("password-repeat")
        if(password != input.value){
            input.setCustomValidity("Password values do not match.")
        }else{
            input.setCustomValidity("")
        }
    }

</script>

结合后端的表单验证

为了好的用户体验，我们往往希望用户在输入用户名的一瞬间，就知道用户名是否可用（而不是提交表单之后）这就要求在用户输入时需要从后端拉取数据，进行表单验证。

实现思路一般是绑定oninput事件，进行ajax请求，根据ajax返回结果，提示用户。
